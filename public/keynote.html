Preemptive Q&A — show, don't tell

"Isn't this just a chatbot / a wrapper around an LLM?"
No chat anywhere in the customer journey. A deterministic 8-signal scorer decides what to recommend in ~70ms; LLMs only write the voice and read the camera. The agent senses (camera + taps + cart), plans (strategy per slot), acts (7 journey surfaces + popup), and verifies (logged counterfactual).SHOW → the system diagram: the agent-loop ribbon lights OBSERVE → HYPOTHESIZE → DECIDE → ACT → VERIFY as it works.
"How do you know it's +15%? Is the number honest?"
Stratified baseline orders vs AI-assisted orders, logged as a counterfactual on production. Every impression and accept is written to D1 — and ops-view probes are excluded so the panel can't inflate acceptance. 78 automated assertions run against the live URL, several of them guarding metric honesty.SHOW → run node test/api-test.mjs <prod> live if they want blood.
"Privacy — you're pointing a camera at customers?"
One coarse glance: age band, attire, group size. No identity, no face matching, no photo storage — the hypothesis literally says "guess" with a confidence bar, and it dies with the session. Behavioral signals alone work fine if the camera is off.SHOW → the hypothesis panel wording + confidence meter.
"Aren't the red badges and the popup manipulative?"
The opposite — they're honesty-constrained: only real POS counts and live promos may write customer-facing copy; margin and inventory motives are whitelisted out and E2E-tested. The popup's struck price must be satisfiable by that dish alone, one popup per session, and anything ignored twice rests (no-nag).SHOW → tests "badges never expose margin/inventory motives" + "spotlight strike price is honest".
"What if the LLM is slow or down?"
The UX survives total LLM outage: pitches race a 1.2s timeout into deterministic sensory copy, the scorer is pure math, vision has a fallback model. This actually happens live sometimes — nothing visibly breaks.
"Show me the decoy actually working."
Open any combo → CUSTOMIZE → tap Khoai Đại — the event stream prints "decoy landed: asymmetric dominance". Three tiers, the middle one exists to make the big one look rational.
"How do stores differ? Does this scale to 250?"
Each store belongs to a cluster (mall/office/residential/tourist) with its own co-occurrence and popularity tables, live inventory, and holiday calendar. One Worker + one D1 serves them all — per-store context is data, not deployment.SHOW → /admin → AI Gợi ý → Chạy thử: Landmark 81/lunch vs Vincom/dinner — visibly different slates.
"Where does the data come from today / how would real POS plug in?"
Today: 9,000 synthetic baskets shaped by cluster × daypart. The engine reads plain co-occurrence/popularity tables — a nightly POS export lands in the same schema. The menu is already real: 106 items crawled live from kfcvietnam.com.vn by TinyFish agents.
"What does it cost at scale?"
~13₫ (~$0.0005) per customer session at Cloudflare list prices — itemized live in the cost panel (gpt-oss $0.35/$0.75 per M tokens, D1 $0.001/M reads…). At 250 stores × 500 sessions/day that's ~$60/day of AI for a measured +28,000₫ per order uplift.
"Human in the loop?"
The P4 ordering agent has a handoff_to_human tool that routes to the first available CS staff; /admin → Hỗ trợ shows the queue with live takeover. The agent knows what it shouldn't decide.
"What happens with a second, different customer?"
Finish the order → BẮT ĐẦU ĐƠN MỚI → inject photo ② (man in a suit). Different persona, different bias bars, different dishes in the live window — same journey.
"Why Cloudflare Workers AI instead of the OpenAI API?"
Everything lives at the edge in one deployment: inference, D1, static assets, telemetry — no cross-cloud latency, one bill, and gpt-oss-120b on Workers AI is fast enough (2–7s agent turns, 1.2s-raced pitches). It also makes the cost meter exact: one provider's list prices.
