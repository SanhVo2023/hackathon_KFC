## 4. Submission requirements (official rules)
Every submission MUST include:
1. **Project description** — problem, solution, how it works, who it's for
2. **Demo** — video walkthrough strongly recommended, and/or live demo URL
3. **AI documentation** — which Agentic AI model(s)/tool(s) used and how they work within the solution (agentic AI must be a CORE component, not a supporting feature)
4. **Source code** — GitHub or equivalent, accessible to judges
5. **Confirmed track**

Also: specify your **tech stack** in the project profile — bonus prizes from technology partners are tied to specific tools.

Rules highlights:
- Team 1–6 members; each person on only one team.
- Projects must be built during the hackathon period (Jul 11 onwards). Pre-existing projects only if Build Week work is a substantial, clearly documented new enhancement.
- Participants retain full IP ownership; sponsors get evaluation rights for pilot conversations.
- Disqualification: no rep at Demo Day, late submission, undocumented pre-existing work, plagiarism, ToS violations, code-of-conduct violations.
- "Make your submission easy to understand. A strong idea with a weak submission will lose to a clear project that judges can evaluate quickly."

## 5. Judging (Demo Day, Jul 12)
Evaluated for **production-readiness, not demo polish**. Dimensions:
1. Technical implementation
2. Quality of solution to the enterprise problem
3. Feasibility of real-world deployment
4. Business impact
5. Pitch clarity

Format:
- **Round 1:** all teams pitch in parallel by track — **3 min pitch + 1 min Q&A**
- **Round 2:** top 5 per track — **7 min pitch + 3 min Q&A**
- Showcase: top team per track on main stage → Awards Ceremony
- Judges include reps from AWS, KFC Vietnam, Phong Vu, VNG Games, Tasco, Vietjet, Google, Techcombank, Tencent Cloud, etc. Full list in portal.

## 7. Best practices (from organizers' FAQ)
- Create project in Event Portal, add all team members.
- Set up all tool accounts and API keys in advance — "Day 4 is for building, not reading documentation."
- Meet the KFC and Phong Vu enterprise reps in person TODAY and get clarifications on P1–P4 before building.
- Prepare a rough pitch structure early; pitch clarity is a judging dimension.
- Submit well before 9:00 AM — do not leave it for the morning of Demo Day. Keep backups of everything (demo build, deck).
- WiFi congestion expected (2,000+ builders) — bring a local-SIM mobile hotspot as backup for API calls/cloud deploys.
- Bring laptop + charger; venue open overnight tonight.
- Builder Experience Track anti-patterns the organizers explicitly disqualify (a signal of what judges dislike generally): chatbot wrappers, static prompts, clones/repackaged tools, mockups without demo, relabeled chat UIs.
